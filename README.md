# 금리와 주가, 그리고 머신러닝

영남대학교 SW중심대학사업단 「AI·SW 러닝메이트 멘토링」
주제: 중앙은행의 금리 정책 기대가 주가에 미치는 영향

이 저장소는 머신러닝으로 주가 방향을 예측해 보면서 "시장은 정말 예측할 수 있는가"라는
질문을 데이터로 따져보는 연구 프로젝트입니다. 프로그래밍이 처음이어도 따라올 수 있도록
코드의 주석과 출력은 모두 한국어로 되어 있습니다.


## 이 프로젝트가 하려는 것

우리가 만드는 도구는 실제 투자회사가 쓰는 것과 크게 다르지 않습니다. 다만 목표가 다릅니다.
돈을 버는 것이 아니라, **시장이 왜 이렇게 예측하기 어려운지를 데이터로 확인하는 것**이 목적입니다.

그래서 몇 가지 원칙을 둡니다.

- 정확도가 낮게 나와도 실패가 아닙니다. 오히려 정직한 결과입니다.
- 정확도가 지나치게 높으면(예: 90%) 프로그램에 실수(데이터 누수)가 있을 가능성이 큽니다.
- "예측이 어렵다"는 결론도 의미 있는 발견입니다. 경제학의 효율적 시장 가설과 일치하기 때문입니다.

모르는 용어가 나오면 아래 [용어 정리](#용어-정리)를 참고하세요.


## 파일 구성

| 단계 | 파일 | 하는 일 | 상태 |
|---|---|---|---|
| 1-a | `phase1a_model_comparison.py` | 네 종류의 모델로 "내일 오를까 내릴까"를 예측하고 성적 비교 | 완료 |
| 1-b | `phase1b_backtest.py` | 그 예측대로 투자했다면 실제로 수익이 났을지 모의실험 | 완료 |
| 1-c | `phase1c_garch_volatility.py` | 금리 발표일에 시장이 더 출렁이는지 변동성 분석 | 완료 |
| 2 | `phase2_event_study.py` | 연준 성명서의 논조(매파/비둘기)와 주가 반응 분석 | 완료 |
| 3 | `phase3_cross_section.py` | 여러 종목 중 다음 달 오를 종목의 순위 예측 | 완료 |

1단계(1-a, 1-b)만 제대로 끝내도 하나의 완결된 발표가 됩니다. 2·3단계는 여유가 있을 때
붙이는 확장입니다. 한 번에 다 하려 하기보다 하나씩 검증하며 진행하는 것을 권합니다.

Colab은 컴퓨터에 아무것도 설치하지 않고 브라우저에서 파이썬을 실행할 수 있는 무료 도구입니다.

1. [Google Colab](https://colab.research.google.com)에 접속해 새 노트를 만듭니다.
2. 실행할 파일(예: `phase1a_model_comparison.py`)의 내용 전체를 복사해 코드 칸에 붙여넣습니다.
3. 맨 위 설치 줄의 `#`을 지우고 한 번 실행합니다. 필요한 도구를 자동으로 설치합니다.

5. 실행하면 표와 그래프가 차례로 나옵니다. LSTM이 포함된 1-a는 몇 분 걸릴 수 있습니다.

## 용어 정리

| 용어 | 쉬운 설명 |
|---|---|
| 정확도 (Accuracy) | 100번 예측해서 몇 번 맞혔는지의 비율. 55번 맞히면 55%. |
| 기준선 (Baseline) | 아무 생각 없이 찍었을 때의 점수. 오르는 날이 더 많으면 "무조건 오른다"만 반복해도 어느 정도 맞는다. 우리 모델은 이 점수를 넘어야 의미가 있다. |
| 정밀도 (Precision) | 모델이 "오른다"고 한 것 중 실제로 오른 비율. |
| 재현율 (Recall) | 실제로 오른 날 중 모델이 잡아낸 비율. |
| F1 점수 | 정밀도와 재현율을 균형 있게 합친 점수. |
| 혼동행렬 (Confusion Matrix) | 맞힌 것과 틀린 것을 네 칸짜리 표로 정리한 것. |
| ROC-AUC | 오를 날과 내릴 날을 얼마나 잘 구별하는지. 0.5면 찍기 수준, 1.0이면 완벽. 정확도에 잘 속지 않아 우리의 주 지표로 쓴다. |
| MCC / 균형정확도 | 한쪽으로 치우친 데이터에서도 잘 속지 않는 지표. MCC가 0, 균형정확도가 0.5면 찍기 수준. |
| 상승예측비율 | 모델이 "오른다"고 답한 비율. 0.9를 넘으면 예측이 아니라 다수결을 흉내내는 중이라는 신호. |
| 데이터 누수 (Data Leakage) | 미래 정보가 학습에 새어드는 것. 시험 문제를 미리 본 것과 같아 점수가 가짜로 높아진다. |
| 과적합 (Overfitting) | 연습문제만 외워서 처음 보는 문제는 틀리는 것. |
| 정상성 (Stationarity) | 값의 성질이 시간이 지나도 크게 변하지 않는 것. 그래서 주가 자체 대신 "평균 대비 몇 %"를 쓴다. |
| 예측 시계 (HORIZON) | 며칠 뒤를 맞힐지. 1이면 내일, 20이면 약 한 달 뒤. |
| 하이퍼파라미터 | 모델이 스스로 배우는 값이 아니라, 사람이 미리 정해주는 설정값. |
| 백테스트 (Backtest) | 과거 데이터로 "이 전략을 썼다면 어땠을까"를 돌려보는 모의실험. |
| 거래비용 | 주식을 사고팔 때마다 떼이는 수수료. 자주 거래하면 무시할 수 없는 크기가 된다. |
| 샤프지수 (Sharpe) | 위험(출렁임) 대비 수익. 높을수록 안정적으로 번 것. |
| 최대낙폭 (MDD) | 고점 대비 가장 크게 떨어진 폭. |
| 변동성 (Volatility) | 가격이 얼마나 심하게 오르내리는지. |
| 매파 / 비둘기 | 매파(hawkish)는 금리를 올리자는 긴축, 비둘기(dovish)는 금리를 내리자는 완화. |
| 생존 편향 | 살아남은 것만 보고 판단하는 착각. 폐업한 가게를 빼고 "이 동네 장사 잘된다"고 말하는 것과 같다. |


## 결과를 읽는 법

정확도 하나만 보면 안 됩니다. 아래를 함께 보아야 합니다.

1. **AUC를 먼저 본다.** 정확도는 상승일이 많으면 "무조건 오른다"만 찍어도 높아져 속기 쉽다.
   AUC는 여기에 잘 속지 않는다. 0.5면 판별력이 전혀 없다는 뜻이다.
2. **상승예측비율을 확인한다.** 모델이 답의 90% 이상을 "상승"으로 찍고 있으면 예측이 아니라
   다수결을 흉내내는 것이다. 코드가 이 경우 자동으로 경고를 출력한다.
3. **정확도와 기준선을 비교한다.** `정확도-기준선`이 양수여야 의미가 있다. `p값(vs기준선)`은
   이것을 통계로 검정한다. 비교 대상은 동전 던지기(0.5)가 아니라 기준선이어야 한다.
4. **예측 기간이 길수록 정확도가 올라 보여도** 기준선도 함께 올랐을 수 있으므로,
   표의 `(기준선 대비)` 값으로 판단한다.
5. **백테스트에서 수익이 났어도** 거래비용을 빼면 사라질 수 있고, 과거 수익이 미래를
   보장하지 않는다.

참고로, 우리도 처음 실제 데이터로 돌렸을 때 이 함정에 빠질 뻔했습니다. 정확도 0.567에 p값이
0.0001로 나와 유의해 보였지만, 확인해 보니 모델이 시험 구간의 97%를 그냥 "상승"으로 찍고
있었고 AUC는 정확히 0.500(판별력 없음)이었습니다. 그래서 검정 기준을 동전 던지기(0.5)가
아니라 기준선으로 바꾸고, AUC를 주 지표로 삼았습니다. 지금 코드는 이 함정을 자동으로 잡아냅니다.


## 검증하려는 가설

아래는 결론이 아니라 검증할 질문입니다. 데이터가 지지할 수도, 반박할 수도 있으며 어느 쪽이든
그대로 보고합니다. 가설에 맞추려고 실험 설정을 바꾸지 않습니다.

| # | 가설 | 이 가설이 틀렸다고 볼 신호 |
|---|---|---|
| H1 | 모델마다 정확도·속도·해석 용이성에 장단점이 있다 | 한 모델이 세 가지 모두 최고다 |
| H2 | 금리 정보를 넣어도 정확도가 크게 오르지 않는다 (효율적 시장) | 금리를 넣으니 정확도가 뚜렷이 오른다 |
| H3 | 금리 '수준'이 중요해 보이는 건 사실 '시기'를 알려주는 것이다 | 시기를 통제해도 금리 수준이 계속 중요하다 |
| H4 | 거래비용을 물면 전략이 매수 후 보유를 못 이긴다 | 비용을 빼고도 전략이 확실히 낫다 |
| H5 | 금리 발표일에는 시장이 평소보다 더 출렁인다 | 발표일과 평소가 통계적으로 차이 없다 |
| H6 | 비둘기 발표 뒤 주가가 매파 발표 뒤보다 더 오른다 | 두 경우의 차이가 통계적으로 의미 없다 |
| H7 | 종목 순위를 맞히는 것이 지수 방향 맞히기보다 유용하다 | 순위 전략이 코스피를 못 이긴다 |

H2는 지지되든 반박되든 둘 다 의미 있는 결과입니다. 지지되면 시장이 효율적이라는 증거이고,
반박되면 금리에 예측 정보가 남아 있다는 더 흥미로운 발견입니다.


## 지키는 원칙

1. 주가·금리를 그대로 쓰지 않는다. 값의 범위가 시기마다 달라지므로(정상성), "평균 대비 몇 %",
   "며칠 동안 얼마 변했나" 같은 비율·변화량으로 바꿔 쓴다.
2. 기준선을 항상 함께 보고한다.
3. 데이터를 시간 순서대로 나눈다. 무작위로 섞으면 미래가 과거로 새어든다(데이터 누수).
4. 구할 수 없는 데이터는 지어내지 않고, 그 분석을 하지 않는다.
5. 난수 시드를 고정해 같은 코드가 같은 결과를 내도록 한다.


## 유의 사항

- 이 코드는 교육·연구용입니다. 실제 투자 판단에 사용하지 마십시오.
- 백테스트 결과가 좋아도 미래 수익을 보장하지 않습니다.
- 낮더라도 정직한 결과가, 부풀린 높은 점수보다 가치 있습니다.
