## 📖 용어 사전 (겁먹지 마세요)

전문 용어가 나와도 아래처럼 쉽게 생각하면 됩니다.

| 용어 | 쉽게 말하면 |
|---|---|
| **정확도 (Accuracy)** | 100번 예측해서 몇 번 맞혔나. 55번 맞히면 55%. |
| **기준선 (Baseline)** | "아무 생각 없이 찍기"의 점수. 예를 들어 오르는 날이 더 많으면 "무조건 오른다"만 외쳐도 어느 정도 맞아요. **우리 AI는 이 점수를 넘어야 "잘한 것"입니다.** (시험에서 다 3번으로 찍은 점수라고 생각하세요) |
| **정밀도 (Precision)** | AI가 "오른다"고 한 것 중, 진짜 오른 비율. (늑대라고 외친 것 중 진짜 늑대 비율) |
| **재현율 (Recall)** | 진짜 오른 날 중, AI가 알아챈 비율. (진짜 늑대가 왔을 때 몇 번 알아챘나) |
| **F1 점수** | 정밀도와 재현율을 균형 있게 합친 점수. |
| **혼동행렬 (Confusion Matrix)** | 맞힌 것/틀린 것을 4칸짜리 표로 정리한 것. |
| **ROC-AUC** | AI가 오를 날과 내릴 날을 얼마나 잘 구별하나. 0.5=찍기 수준, 1.0=완벽. |
| **데이터 누수 (Data Leakage)** | 시험 문제를 미리 훔쳐본 것과 같은 반칙. 점수가 가짜로 높아져요. 🚨 |
| **과적합 (Overfitting)** | 연습문제만 달달 외워서, 처음 보는 실전 문제는 틀리는 것. |
| **정상성 (Stationarity)** | 값의 성질이 시간이 지나도 크게 안 변하는 것. 그래서 주가 자체 대신 "평균보다 몇 % 위인가"를 씁니다. |
| **예측 시계 (HORIZON)** | 며칠 뒤를 맞힐지. 1이면 내일, 20이면 한 달 뒤. |
| **하이퍼파라미터** | AI가 스스로 배우는 게 아니라, 사람이 미리 정해주는 설정값. |
| **백테스트 (Backtest)** | 과거 데이터로 "만약 이 전략으로 투자했다면?"을 돌려보는 모의실험. |
| **거래비용** | 주식을 사고팔 때마다 떼이는 수수료. 티끌 같아도 자주 거래하면 큰돈. |
| **샤프지수 (Sharpe)** | 위험(출렁임) 대비 수익. 높을수록 마음 편히 번 돈. |
| **최대낙폭 (MDD)** | 고점 대비 가장 크게 떨어졌던 폭. 얼마나 마음 졸였는지의 지표. |
| **변동성 (Volatility)** | 가격이 얼마나 심하게 오르내리나. |
| **매파 / 비둘기** | 매파(Hawkish)=금리 올리자(긴축), 비둘기(Dovish)=금리 내리자(완화). |
| **생존 편향** | 살아남은 것만 보고 착각하는 것. (망한 가게는 빼놓고 "이 동네 장사 잘돼!"라고 하는 것) |

## 🔎 결과를 어떻게 읽나요? (가장 중요!)

**"정확도 몇 %"만 보면 안 돼요.** 아래 4가지를 꼭 같이 보세요.

1. **정확도 vs 기준선** — 표의 `정확도-기준선`이 **플러스(+)** 여야 AI가 의미 있게 잘한 거예요. 마이너스면 "찍기보다 못한 것".
2. **`p값`이 0.05보다 크면** — "이 성적은 우연일 수도 있다"는 뜻. 나쁜 게 아니라 정직한 신호예요.
3. **예측 기간이 길수록 정확도가 올라 보여도** — 기준선도 같이 오른 것일 수 있어요. 그래서 표에 `(기준선 대비)`를 함께 적어 뒀어요. 그걸로 판단하세요.
4. **백테스트에서 돈을 벌었어도** — 거래비용을 빼면 사라질 수 있어요. 그리고 과거에 벌었다고 미래에도 번다는 보장은 **절대 없습니다.**

각 단계별 자세한 해석은 파일을 실행하면 **한국어 힌트**가 함께 출력됩니다.

## 🧪 우리가 확인하는 가설 (정답이 아니라 '검증할 질문')

아래는 결론이 아니라 **검증할 질문**이에요. 데이터가 "맞다" 할 수도, "틀리다" 할 수도 있어요. 어느 쪽이든 그대로 보고합니다. 가설에 맞추려고 실험을 바꾸면 안 돼요!

| # | 가설 | 이 가설이 **틀렸다**고 볼 신호 |
|---|---|---|
| H1 | 모델마다 정확도·속도·이해하기 쉬움에 장단점이 있다 | 한 모델이 셋 다 최고다 |
| H2 | 금리 정보를 넣어도 정확도가 크게 안 오른다 (시장이 효율적) | 금리를 넣으니 정확도가 뚜렷이 오른다 |
| H3 | 금리 '수준'이 중요해 보이는 건 사실 '시기'를 알려주는 것뿐이다 | 시기를 제거해도 금리 수준이 계속 중요하다 |
| H4 | 거래비용을 물면 전략이 그냥 사서 들고 있기를 못 이긴다 | 비용을 빼고도 전략이 확실히 더 낫다 |
| H5 | 금리 발표일엔 시장이 평소보다 더 출렁인다 | 발표일과 평소가 통계적으로 차이 없다 |
| H6 | 비둘기 발표 뒤 주가가 매파 발표 뒤보다 더 오른다 | 두 경우의 차이가 통계적으로 의미 없다 |
| H7 | 종목 순위를 맞히는 게 지수 방향 맞히기보다 유용하다 | 순위 전략이 코스피를 못 이긴다 |

> 💡 H2가 맞아도 틀려도 둘 다 좋은 결과예요. 맞으면 → "시장은 효율적"이라는 증거. 틀리면 → "금리에 예측 힌트가 숨어 있다"는 더 흥미로운 발견!

## ✅ 우리가 꼭 지키는 규칙

1. **주가·금리를 그대로 안 쓰기** — 값이 시대마다 크게 달라지니까(정상성), "평균 대비 몇 %", "며칠 새 얼마 변했나"로 바꿔 씁니다.
2. **기준선을 항상 같이 보기** — 찍기 점수를 넘는지 확인.
3. **시간 순서 지키기** — 데이터를 섞지 않아요. 미래가 과거로 새어들면 반칙(데이터 누수).
4. **데이터를 지어내지 않기** — 못 구하는 데이터는 그냥 분석 안 해요.
5. **똑같은 결과가 나오게 하기** — 난수 시드를 고정(`42`).
